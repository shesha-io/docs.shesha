## Project Overview

This is the official Shesha Framework documentation site ([docs.shesha.io](https://docs.shesha.io/)), built with **Docusaurus v3** (classic preset). It produces a static site deployed to Azure Web Apps.

## Build & Development Commands

```bash
npm install          # Install dependencies (requires Node >=18.0)
npm start            # Local dev server with hot reload
npm run build        # Production build to /build directory
npm run serve        # Serve built site locally (required to test search)
npm run clear        # Clear Docusaurus cache
```

There are no linting or test commands configured.

## Architecture

### Documentation Structure

All docs live in `docs/` with autogenerated sidebars (see `sidebars.js`). Category ordering is controlled by `_category_.json` files in each directory. The main sections are:

1. **get-started/** — Introduction, FAQ, tutorials
2. **back-end-basics/** — Backend development guides
3. **front-end-basics/** — Frontend guides, form components, JS API
4. **fundamentals/** — Core concepts, security
5. **how-to-guides/** — Task-oriented guides
6. **manage-apps-and-users/** — Admin workflows
7. **settings/** — Configuration reference
8. **for-administrators/** — Deployment and admin topics
9. **framework-contributors/** — Contributing to Shesha itself

### Custom Components (`src/`)

- **`src/components/ask-ai/`** — AI-powered search modal with chat history and markdown rendering (AISearch.jsx is the main component)
- **`src/components/custom-search/`** — Search bar that toggles between AI search and standard Docusaurus search
- **`src/components/docs/`** — Reusable doc components (Banner, Card using Ant Design)
- **`src/components/theme/`** — Docusaurus theme overrides (DocSidebar, NavbarItem)
- **`src/pages/index.js`** — Redirects root `/` to `/docs/get-started/Introduction`
- **`src/css/custom.css`** — Global site styles

### Key Configuration

- **`docusaurus.config.js`** — Site config, navbar, footer, analytics (Google Tag Manager, Hotjar), search plugin (docusaurus-lunr-search), C# syntax highlighting via Prism
- **`sidebars.js`** — Uses `type: 'autogenerated'` from the `docs/` directory
- **`static/scripts/`** — Analytics event tracking scripts injected via config

### CI/CD

- **`shesha-docs-BP.yaml`** — Azure Pipelines: triggers on `main` and PRs, builds on Windows with Node 18.15.0, produces a zip artifact
- **`ReleaseFlow.ps1`** — PowerShell script for semantic version calculation from branch/tag names

## Conventions

- Documentation files are Markdown (`.md`) or MDX (`.mdx`)
- Images and videos go in `static/img/` and `static/vids/`
- Light mode only (dark mode toggle is disabled in config)
- Ant Design (`antd` v5) is available for custom React components
- The site uses `docusaurus-lunr-search` for offline full-text search
